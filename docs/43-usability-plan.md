# Usability Plan

## Overview

This usability plan outlines the approach for evaluating the **MyChild Helpline** app. The evaluation aims to determine whether users can efficiently navigate the app, complete common tasks, and access key features with minimal difficulty.

## Objectives and Scope

### Objectives

The usability evaluation aims to:

* Evaluate the ease of navigation within the app.
* Assess whether users can successfully complete common tasks.
* Identify usability issues and opportunities for improvement.

### Scope

The evaluation covers the following features:

* Language selection
* User profile setup
* Navigation throughout the app
* Access to informational resources
* Access to support services

## Test Scenarios and Procedures

### Test Scenarios

Participants will be asked to complete the following tasks:

1. Launch the **MyChild Helpline** app.
2. Select their preferred language.
3. Complete the user profile setup.
4. Navigate to a specified feature or information page.
5. Locate available support resources.

### Test Procedure

During each usability session:

1. Introduce the participant to the evaluation.
2. Explain the assigned tasks without providing assistance.
3. Observe the participant's interactions while recording any issues or difficulties encountered.
4. Collect feedback after the participant completes the tasks.

## Evaluation Metrics and Reporting

### Evaluation Metrics

The following measures will be collected during testing:

* Task completion rate
* Time required to complete each task
* Number of errors encountered
* User satisfaction and qualitative feedback

### Reporting

The results of the usability evaluation will be documented in a report that includes:

* A summary of test findings
* Identified usability issues
* Recommendations for future improvements
