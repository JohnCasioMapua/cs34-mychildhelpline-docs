# MyChild Helpline App Quick-Reference Style Guide

## Purpose & tone

### Purpose

This style guide establishes consistent writing standards for all MyChild Helpline App documentation, including user manuals, help guides, FAQs, and support materials.

### Tone and voice

Documentation should be:

- Professional
- Clear and concise
- Supportive and reassuring
- User-focused
- Easy to understand for users with varying levels of technical knowledge

When discussing mental health and support services, use respectful, empathetic, and non-judgmental language.

### Active vs. passive voice

Use **active voice** whenever possible.

**Preferred:**

- Tap **Live Chat** to connect with a counsellor.
- Open **Mood Tracker** to record your emotions.

**Avoid:**

- The **Live Chat** feature can be accessed by tapping the button.
- Emotions can be recorded using the **Mood Tracker**.

## Terminology & word choice

### Technical jargon

Avoid unnecessary technical jargon. Use simple language that users can easily understand.

**Preferred:**

- Sign in
- Tap
- Account
- Support service

**Avoid:**

- Authenticate
- Execute
- User credential repository
- Intervention platform

### Acronym management

Spell out acronyms on first mention followed by the acronym in parentheses.

**Examples:**

- Frequently Asked Questions (FAQ)
- Application Programming Interface (API)
- Content Management System (CMS)

After the first mention, the acronym may be used alone.

### Preferred terminology

| Preferred Term | Discouraged Term |
|----------------|------------------|
| Child | Kid |
| User | Client |
| Counsellor | Operator |
| Support service | Intervention |
| Sign in | Log into |
| Tap | Click |
| Mental health support | Psychological intervention program |

### Mini-glossary

| Term | Definition |
|------|------------|
| Helpline | A support service that users can contact for assistance. |
| Live Chat | A feature that allows users to communicate with a counsellor in real time. |
| Mood Tracker | A tool used to record and monitor emotions. |
| Journal | A private space where users can record thoughts and experiences. |
| Counsellor | A trained professional who provides support and guidance. |

## Markdown formatting rules

### Bold text

Use **bold text** only for:

- Buttons
- Menu names
- Screen names
- Important notices or warnings

**Examples:**

- Tap **Live Chat**.
- Select **Mood Tracker** from the main menu.
- **Important:** Ensure your internet connection is stable before starting a live chat session.

### Code formatting (backticks)

Use inline backticks only for:

- File names
- Folder names
- Commands
- Database fields
- Variable names

**Examples:**

- Open `README.md`.
- Update the `user_profile` record.
- Run `npm install`.

### Code blocks

Use fenced code blocks for multi-line commands or code examples.

```bash
npm install
npm start
```

### Heading levels

Use headings in the following structure:

```markdown
# Document title
## Main section
### Subsection
#### Supporting detail
```

Rules:

- `#` = Document title only
- `##` = Major sections
- `###` = Subsections
- `####` = Supporting details

Do not skip heading levels.

## Punctuation & mechanics

### Lists

Rules:

- Start each bullet point with a capital letter.
- Use parallel sentence structure.
- End complete-sentence bullet points with a period.
- Do not use periods for short fragments.

**Example:**

- Open the application.
- Sign in to your account.
- Select **Live Chat**.

### Oxford comma

Use the Oxford comma in all documentation.

**Correct:**

- The application includes Live Chat, Mood Tracker, and Journal features.

**Incorrect:**

- The application includes Live Chat, Mood Tracker and Journal features.

### Headings

Use **sentence case** for all headings.

**Correct:**

- Creating a new account
- Using the mood tracker

**Incorrect:**

- Creating A New Account
- Using The Mood Tracker

## Do's and don'ts

| Don't | Do |
|-------|----|
| Click on the Live Chat button. | Tap **Live Chat**. |
| The journal can be accessed by the user through the menu. | Open **Journal** from the menu. |
| Utilize the Mood Tracker feature. | Use **Mood Tracker** to record your emotions. |
| Login to your account. | Sign in to your account. |
| The support service is available for use by users. | Access support services through the app. |

---

## Style guide checklist

Before publishing documentation, verify that:

- Active voice is used whenever possible.
- Mental health-related content uses respectful and supportive language.
- Acronyms are defined on first mention.
- Headings follow the approved hierarchy.
- Bold text is used only for UI elements and warnings.
- Backticks are used only for technical references.
- Lists follow capitalization and punctuation rules.
- The Oxford comma is used consistently.
- Approved terminology is used throughout the document.
